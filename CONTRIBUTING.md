# Contributing to my-ford-benefits


Thank you for your interest in contributing to **my-ford-benefits**! We appreciate code, documentation, bug reports, feature suggestions, and tests. To keep contributions smooth, please follow the guidelines below.


## Contribution Guidelines
- Follow the repository coding standards (use consistent formatting, descriptive names).
- Write clear, small commits and include a helpful commit message.
- Add unit tests for new logic when applicable.
- Keep pull requests focused (one logical change per PR).


## How to Contribute
1. Fork the repository.
2. Create a feature branch: `git checkout -b feat/my-change`.
3. Make your changes.
4. Run tests and linters locally.
5. Commit and push your branch to your fork.
6. Open a Pull Request against the `main` branch of this repo.


## Reporting Issues
If you find a bug or an inconsistency, please open an issue with:
- A descriptive title
- Steps to reproduce
- Expected and actual behavior
- Any relevant screenshots or logs

## Code of Conduct
Be respectful and kind. We do not tolerate harassment or discrimination. Keep discussions constructive. Violations may result in removal from the project.


## Review Process
- Pull requests will be reviewed by at least one maintainer.
- Reviews will focus on correctness, readability, tests, and security.
- Please respond to review comments and update your PR.
- Once approved, a maintainer will merge your PR.


Thank you for helping improve **my-ford-benefits**!
